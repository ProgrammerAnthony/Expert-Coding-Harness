# 实施计划审查子代理提示词模板

在完整计划写出后、交付用户前，可选派发「计划文档审查」子代理时使用。

**目的：** 确认计划完整、与规格对齐、任务可执行。

```
Task 工具（通用 / general-purpose）:
  description: "审查实施计划文档"
  prompt: |
    你是实施计划审查员。确认该计划已可交付执行。

    **待审查计划：** [PLAN_FILE_PATH]
    **对照规格：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注 |
    |------|------|
    | 完整性 | TODO、占位、缺步骤、无验证命令 |
    | 与规格一致 | 是否覆盖规格要求、有无明显范围蔓延 |
    | 任务分解 | 任务边界是否清晰、步骤是否可执行 |
    | 可构建性 | 工程师/代理按文档能否推进而不卡死 |

    ## 校准

    **只标记会在实施中造成实质问题的事项。**
    会导致做错事或无法继续的才算问题；轻微措辞、风格建议不阻塞通过。

    除非存在严重缺口（规格需求未覆盖、步骤矛盾、大量占位、任务空泛），否则应予以通过。

    ## 输出格式

    ## 计划审查

    **结论：** 通过 | 有问题

    **问题（如有）：**
    - [任务 X 步骤 Y]：具体问题 — 对实施的影响

    **建议（可选，不阻塞通过）：**
    - …
```

**审查员返回：** 结论、问题列表、建议。
