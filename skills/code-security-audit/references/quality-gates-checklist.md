## 通用质量门控清单（可复制到输出）

### 可执行性

- [ ] 每个步骤都包含明确的**路径/命令/预期输出**
- [ ] 没有 `TODO/TBD/适当/类似/等等/后续再说` 这类占位词
- [ ] 关键决策点（分支策略、范围取舍、破坏性变更）已显式列出并等待用户确认

### 证据要求

- [ ] 结论都有证据：引用文件路径/行号/日志/命令输出
- [ ] 不基于“典型框架通常如此”做断言；不确定必须标为“需人工验证”

### 范围与风险

- [ ] 明确 Out of Scope，避免范围蔓延
- [ ] 标注高风险变更（数据写入、权限边界、外部请求、加密、删除/迁移）

### 验收与验证

- [ ] 有可测试的验收标准（可用命令验证）
- [ ] 修改后运行了最小回归验证（至少覆盖关键路径）
