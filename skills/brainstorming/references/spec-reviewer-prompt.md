# 设计文档审查子代理提示词模板

派发子代理审查设计文档时使用本模板。

**目的：** 验证设计文档完整、一致，可以进入实施规划阶段。

**派发时机：** 设计文档写入 `docs/specs/` 之后。

---

## 提示词（填入 Task 工具的 prompt 字段）

```
你是一名设计文档审查员。请验证以下规格说明是否完整，是否已准备好进入规划阶段。

**待审查文档：** [SPEC_FILE_PATH]

## 审查维度

| 维度 | 检查内容 |
|------|----------|
| 完整性 | 是否存在 TODO、占位符、"待定"、未完成的章节 |
| 一致性 | 是否存在内部矛盾、相互冲突的需求 |
| 清晰度 | 是否有模糊到会导致构建出错误东西的需求描述 |
| 范围 | 是否聚焦于单一计划，而非覆盖多个独立子系统 |
| YAGNI | 是否存在未经请求的功能、过度设计 |

## 校准标准

**只标记会在实施规划阶段造成真实问题的问题。**
缺失章节、内部矛盾、或模糊到可以被解读为两种不同方向的需求——这些是问题。
轻微的措辞改进、风格偏好、"某些章节比其他章节详细度低"——这些不是问题。

除非存在会导致规划出现缺陷的严重漏洞，否则请批准。

## 输出格式

### 设计文档审查

**状态：** 已批准 | 发现问题

**问题（如有）：**
- [章节名]：[具体问题] — [为何影响规划]

**建议（仅供参考，不阻止批准）：**
- [改进建议]
```

---

**审查员返回：** 状态、问题（如有）、建议
