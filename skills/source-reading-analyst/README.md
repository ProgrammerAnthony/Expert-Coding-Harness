# 源码阅读分析师

面向“看懂现有代码”的结构化技能，覆盖三类高频场景：

- 定位问答：快速回答“功能在哪里、怎么调用、为何这样设计”
- 阅读报告：输出模块职责、调用链、数据流与关键风险
- 改造建议：在只读前提下给出可执行的重构与演进建议

## 安装

```bash
npx skills add ProgrammerAnthony/Expert-Coding-Skills --path skills/source-reading-analyst
```

## 使用方式

在 Agent 终端中输入：

```text
/源码分析
```

或：

```text
/source-reading
```

## 默认行为

- 默认只读分析，不修改代码
- 先定位证据，再给结论
- 所有结论必须绑定文件路径与关键片段

## 输出能力

| 类型 | 输出内容 |
|------|----------|
| 快速问答 | 定位入口、调用路径、关键依赖、风险提示 |
| 全量导读 | 模块地图、职责边界、主流程与异常流 |
| 改造建议 | 分层拆分建议、优先级路线图、验证建议 |

## 注意事项

- 当问题涉及跨模块链路时，先画“文字版调用链”再展开细节
- 遇到信息不足时先标注假设，不可臆断
- 若用户希望改代码，必须先得到明确确认
