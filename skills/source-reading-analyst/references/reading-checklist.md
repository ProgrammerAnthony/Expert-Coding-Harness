# 阅读分析检查清单

用于保证"源码阅读分析"过程完整、可复核。

## 1. 入口定位

- [ ] 是否识别了系统入口（CLI / HTTP / Job / Event）？
- [ ] 是否定位了关键路由或调度点？
- [ ] 是否找到核心初始化流程？

## 2. 调用链梳理

- [ ] 是否给出了主流程调用链？
- [ ] 是否区分了同步链路与异步链路？
- [ ] 是否识别跨模块调用边界？

## 3. 数据流与状态

- [ ] 输入来自哪里（请求/消息/文件/数据库）？
- [ ] 中间状态在哪些对象中流转？
- [ ] 持久化发生在何处（DB/Cache/外部服务）？
- [ ] 是否标注关键字段或状态机转换？

## 4. 异常与鲁棒性

- [ ] 主要错误类型有哪些？
- [ ] 错误如何处理（返回、重试、降级、告警）？
- [ ] 是否存在吞错或静默失败路径？

## 5. 架构与边界

- [ ] 模块职责是否清晰？
- [ ] 是否存在高耦合或循环依赖迹象？
- [ ] 外部依赖边界是否明确？
- [ ] 权限或安全边界是否可识别？

## 6. 可演进性

- [ ] 是否识别到高变更频率区域？
- [ ] 是否识别到可测试性薄弱点？
- [ ] 是否给出低风险改造切入点？

## 7. 结论质量

- [ ] 关键结论是否都附带代码证据？
- [ ] 结论与建议是否区分（事实 vs 观点）？
- [ ] 是否明确标记"待验证项"？

## 8. 图表完整性

- [ ] 是否生成了模块依赖图（`graph TD`）？
- [ ] 是否生成了核心流程序列图（`sequenceDiagram`）？
- [ ] 若存在状态机/流程分支，是否生成了状态图或流程图？
- [ ] 若数据模型复杂，是否生成了 ER 图或类图？
- [ ] 每张图是否有一行中文说明其观察角度？
- [ ] 图中节点标签是否与代码中的实际名称一致？
- [ ] 异步调用是否用虚线与标注文字加以区分？
- [ ] 改造建议是否包含改造前/后对比图？
