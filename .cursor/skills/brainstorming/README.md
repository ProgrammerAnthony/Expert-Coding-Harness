# 方案设计师

在任何创作性工作（新功能、组件构建、行为修改）之前触发的结构化头脑风暴技能，通过协作对话将想法转化为经过验证的设计文档，再进入实施阶段。

## 安装

```bash
npx skills add ProgrammerAnthony/Expert-Coding-Skills --path skills/brainstorming
```

## 使用方式

在 Agent 终端中输入：

```
/方案设计
```

或：

```
/brainstorming
```

## 核心约束

- 在获得用户明确批准前，**禁止编写任何代码**
- 每次只问**一个问题**，不用多个问题淹没用户
- 始终提出 **2-3 个方案**，让用户了解权衡
- 设计文档写完后必须经过**自动审查循环**，再交用户确认

## 工作流

| 阶段 | 内容 |
|------|------|
| 1. 上下文探索 | 检查文件、文档、最近提交 |
| 2. 范围评估 | 判断是否需要拆分子项目 |
| 3. 澄清问题 | 每次一个问题，逐步对齐目的/约束/成功标准 |
| 4. 方案对比 | 提出 2-3 个方案，说明权衡与推荐理由 |
| 5. 设计展示 | 分节展示，每节获得确认后继续 |
| 6. 文档落地 | 写入 `docs/specs/YYYY-MM-DD-<主题>-设计.md` 并提交 |
| 7. 自动审查 | 派发审查子代理，发现问题修复后重审（最多 3 轮） |
| 8. 用户确认 | 等用户审阅设计文档后再推进 |
