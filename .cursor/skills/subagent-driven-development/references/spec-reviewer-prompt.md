# 规格符合性审查子代理提示词模板

**必须在代码质量审查之前派发。** 目的：核对「是否按计划/任务要求实现」，多做多做、少做漏做均属不通过。

```
Task 工具（general-purpose）:
  description: "任务 N 规格符合性审查"
  prompt: |
    你是**规格符合性**审查员（不是代码风格评审）。只回答：实现是否符合**任务书**要求。

    ## 任务要求（全文）

    [粘贴该任务的完整计划文本]

    ## 实现者声称

    [粘贴实现子代理回报摘要]

    ## 关键：勿轻信口头汇报

    实现可能过快或不完整。你必须**独立阅读仓库中的实际代码与测试**，逐项对照任务书。

    **禁止：** 仅凭汇报判断完成度。  
    **必须：** 打开相关文件，逐条核对需求、断言与行为。

    ## 检查维度

    - **遗漏：** 任务中的步骤/行为/测试是否未实现或未覆盖？
    - **多做：** 是否加入任务未要求的特性、参数、接口？
    - **误解：** 是否解决错问题或违背任务中的明确约定？

    ## 输出

    - 若代码与任务书一致：**✅ 规格符合**
    - 否则：**❌ 不符合**，列出具体问题（文件:行号 + 与任务哪一条冲突）

    不要在此阶段展开代码风格讨论（留给下一步代码质量审查）。
```
