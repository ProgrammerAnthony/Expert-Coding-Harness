# 开发工作流

## 功能实现流水线

1. **先设计**
   - 使用 **方案设计师** 技能（`/方案设计`）
   - 识别依赖和风险
   - 拆分为阶段

2. **TDD 方式**
   - 使用 **TDD 开发大师** 技能（`/tdd`）
   - 先写测试（RED）
   - 最小化实现使测试通过（GREEN）
   - 重构（IMPROVE）
   - 验证覆盖率 80%+

3. **代码审查**
   - 写完代码后立即使用 **代码审查专家** 技能（`/代码审查`）
   - 处理 CRITICAL 和 HIGH 问题
   - 尽量修复 MEDIUM 问题

4. **提交与推送**
   - 详细的提交消息
   - 遵循 Conventional Commits 格式
   - 参考 Git 工作流规则

## 可用技能

| 技能 | 触发方式 | 适用场景 |
|------|---------|---------|
| 方案设计师 | `/方案设计` | 实现前的设计与规划 |
| TDD 开发大师 | `/tdd` | 新功能、Bug 修复 |
| 代码审查专家 | `/代码审查` | 写完代码后 |
| 代码安全审计 | `/代码安全审计` | 提交前安全检查 |
| 架构顾问 | `/架构分析` | 架构决策 |
| 调试专家 | `/调试` | 排查 Bug |
