# 测试要求

## 最低测试覆盖率：80%

必须包含的测试类型：
1. **单元测试** — 独立函数、工具函数、组件
2. **集成测试** — API 端点、数据库操作
3. **E2E 测试** — 关键用户流程

## 测试驱动开发（TDD）

强制工作流：
1. 先写测试（RED）
2. 运行测试 — 应当**失败**
3. 最小化实现（GREEN）
4. 运行测试 — 应当**通过**
5. 重构（IMPROVE）
6. 验证覆盖率（80%+）

使用 **TDD 开发大师** 技能（`/tdd`）自动引导此流程。

## 测试失败排查

1. 检查测试隔离
2. 验证 mock 是否正确
3. 修复实现，而非修改测试（除非测试本身有误）

## 禁止的测试反模式

- 跳过失败的测试（`skip`、`xit`）
- 测试中有魔法数字（无意义的常量）
- 测试依赖执行顺序
- 在 mock 上做断言而非真实行为
