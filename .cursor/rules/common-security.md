# 安全规范

## 强制安全检查

提交任何代码前：
- [ ] 无硬编码密钥（API Key、密码、Token）
- [ ] 所有用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（HTML 输出已转义）
- [ ] CSRF 保护已启用
- [ ] 认证/授权已验证
- [ ] 所有端点有频率限制
- [ ] 错误消息不泄露敏感数据

## 密钥管理

- 永不在源码中硬编码密钥
- 始终使用环境变量或密钥管理工具
- 启动时验证所需密钥是否存在
- 轮换任何可能已泄露的密钥

## 安全响应协议

发现安全问题时：
1. 立即停止
2. 使用项目内的 **代码安全审计专家** 技能（`/代码安全审计`）
3. 在继续前修复 CRITICAL 问题
4. 轮换已暴露的密钥
5. 审查整个代码库中的类似问题
